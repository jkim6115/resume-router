# AI 변경 워크플로

이 문서는 사용자가 기능 추가나 수정을 AI에게 요청할 때 따르는 작업 흐름을 정의합니다.

목표는 빠른 개발보다 예측 가능한 변경입니다. 특히 CSS, 관리자 UI, 공개 이력서 레이아웃이 의도치 않게 무너지지 않도록 계획과 검증 단계를 고정합니다.

## 기본 흐름

1. 사용자가 프롬프트로 요구사항을 입력합니다.
2. AI는 바로 개발하지 않고 먼저 계획을 수립합니다.
3. 사용자가 계획을 승인합니다.
4. AI가 개발을 진행합니다.
5. AI가 테스트를 실행하고 결과를 보고합니다.

승인 전에는 파일 수정, 의존성 설치, 스키마 변경, 테스트 스냅샷 갱신을 진행하지 않는 것을 원칙으로 합니다. 단, 현황 파악을 위한 읽기 작업은 가능합니다.

## 1. 프롬프트 입력

사용자는 변경하고 싶은 기능, 화면, 문제를 자연어로 설명합니다.

예시:

```text
관리자 목록에서 기업별 이력서를 상태별로 필터링하고 싶다.
```

좋은 프롬프트에는 다음 정보가 포함됩니다.

- 변경 대상 화면 또는 기능
- 원하는 사용자 경험
- 바꾸면 안 되는 부분
- 디자인 변경 여부
- 데이터 구조 변경 가능 여부

## 2. AI 계획 수립

AI는 요구사항을 받은 뒤 다음 항목을 포함한 계획을 먼저 제안합니다.

- 변경 목적
- 예상 수정 파일
- 데이터 모델 변경 여부
- UI/CSS 영향 범위
- 기존 기능에 대한 위험 요소
- 추가하거나 수정할 테스트
- 실행할 검증 명령어

이 단계에서 테스트 계획도 함께 제안합니다. 테스트는 개발이 끝난 뒤에만 붙이는 것이 아니라, 계획 단계에서 어떤 안전망을 사용할지 먼저 정해야 합니다.

## 3. 사용자 승인

사용자는 계획을 검토한 뒤 승인하거나 수정 요청을 합니다.

승인 예시:

```text
승인
```

수정 요청 예시:

```text
UI 변경은 빼고 데이터와 관리자 기능만 진행
```

사용자가 승인하기 전까지 AI는 개발을 시작하지 않습니다.

## 4. 개발

승인 후 AI는 계획에 따라 개발합니다.

개발 중 지켜야 할 기준:

- 기존 Next.js App Router 구조를 유지합니다.
- 공개 이력서 페이지에는 회사명을 노출하지 않습니다.
- 관리자 UI의 버튼, 테이블, 폼, 저장 버튼 스타일을 임의로 바꾸지 않습니다.
- 전역 CSS 변경은 영향 범위를 먼저 확인합니다.
- 의도적인 디자인 변경이 아니라면 시각 회귀 스냅샷을 갱신하지 않습니다.

## 테스트 배치 기준

테스트는 두 지점에 붙이는 것이 좋습니다.

첫 번째는 계획 단계입니다. AI가 어떤 테스트를 추가하거나 실행할지 계획에 포함해야 합니다. 이때 사용자는 테스트 범위가 과하거나 부족하지 않은지 승인할 수 있습니다.

두 번째는 개발 완료 직후입니다. 최종 보고 전에 테스트를 실행해 실제로 기능과 화면이 유지되는지 확인합니다.

권장 기준:

| 변경 유형 | 테스트 위치 | 권장 테스트 |
| --- | --- | --- |
| 문서만 변경 | 개발 완료 후 | 별도 테스트 생략 가능 |
| 서버 로직 변경 | 개발 완료 후 | `npm run build` |
| Prisma 스키마 변경 | 개발 중/완료 후 | `npm run db:push`, `npm run build` |
| 관리자 기능 변경 | 개발 완료 후 | `npm run build`, `npm run test:e2e` |
| 공개 이력서 변경 | 개발 완료 후 | `npm run build`, `npm run test:e2e`, `npm run test:visual` |
| CSS 또는 레이아웃 변경 | 개발 완료 후 | `npm run build`, `npm run test:e2e`, `npm run test:visual` |
| 의도적인 디자인 변경 | 사용자 승인 후 | `npm run test:visual:update`, 이후 `npm run test:visual` |

기본 검증 루틴:

```bash
npm run build
npm run test:e2e
npm run test:visual
```

Prisma 스키마를 변경한 경우:

```bash
npm run db:push
npm run build
npm run test:e2e
npm run test:visual
```

## 시각 회귀 테스트 운영

`npm run test:visual`은 기존 스크린샷과 현재 화면을 비교합니다. CSS가 깨지거나 버튼 위치, 테이블 구조, 공개 이력서 문서 레이아웃이 달라지면 실패할 수 있습니다.

디자인 변경이 의도된 경우에만 다음 명령으로 기준 스크린샷을 갱신합니다.

```bash
npm run test:visual:update
```

스냅샷 갱신은 반드시 사용자의 승인 후 진행합니다.

## 최종 보고

AI는 작업 완료 후 다음 내용을 간단히 보고합니다.

- 변경한 내용
- 실행한 테스트
- 통과 여부
- 실행하지 못한 테스트가 있다면 그 이유
- 사용자가 확인하면 좋은 화면이나 경로
