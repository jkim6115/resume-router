# CLAUDE.md

이 프로젝트의 AI 개발 워크플로는 3개의 에이전트로 분담됩니다.

## 에이전트 역할

### Claude Code Opus — 계획 담당

사용자 요청을 받으면 Opus가 계획을 수립합니다.

- 코드베이스를 읽고 요구사항을 분석합니다.
- 변경 파일, 데이터 모델 영향, UI 위험 범위, 테스트 범위를 포함한 계획을 출력합니다.
- 계획 단계에서는 파일을 수정하지 않습니다.
- 사용자 승인을 받기 전까지 구현을 시작하지 않습니다.

### Claude Code Sonnet — 구현 담당

사용자가 Opus 계획을 승인한 뒤 Sonnet이 구현합니다.

- 계획에 명시된 파일만 수정합니다.
- 테스트는 실행하지 않습니다.
- 구현 완료 후 Codex 리뷰를 안내합니다.

### Codex (plugin) — 리뷰 담당

Sonnet 구현이 끝나면 `/codex:review`를 실행해 코드 리뷰를 진행합니다.

- 코드 품질, 보안, 패턴 일관성, 버그 가능성을 검토합니다.
- 지적 사항이 있으면 Sonnet이 수정합니다.

## 전체 플로우 요약

```
사용자 요청
  → Opus: 계획 수립
  → 사용자: 승인
  → Sonnet: 구현
  → Codex: 코드 리뷰 (/codex:review)
  → (지적 사항 발생 시) Opus → Sonnet → Codex 반복
  → 최종 보고
```
